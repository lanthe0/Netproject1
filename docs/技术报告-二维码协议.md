## 1. 系统概述

### 1.1 项目背景

本项目实现了一个基于视觉通道的数据传输系统，通过将二进制文件编码为二维码视频，实现屏幕到相机的单向数据传输。该系统适用于：
- 跨设备文件传输（无需网络连接）
- 离线数据备份和恢复
- 安全场景下的物理隔离数据传输

### 1.2 核心特性

✅ **自定义二维码协议**: 144×144网格，优化容量与识别率平衡  
✅ **强纠错能力**: Reed-Solomon(16)编码，理论可纠正8字节错误  
✅ **帧独立设计**: 每帧自包含完整信息，丢帧不影响其他帧  
✅ **高数据密度**: 单帧2,183字节净容量  
✅ **端到端完整性验证**: CRC32校验 + RS纠错双重保护  

### 1.3 系统工作流程

`
[二进制文件] 
    ↓ encode_bin()
[二维码矩阵序列] 
    ↓ bin2video()
[MP4视频文件] 
    ↓ 屏幕播放 → 相机拍摄
[视频帧序列] 
    ↓ video_to_qr_sequence()
[二维码识别] 
    ↓ decode_image()
[恢复的二进制文件]
`

## 2. 协议设计

### 2.1 二维码结构 (144×144 Grid)

我们的二维码采用自定义设计，而非标准QR Code，以优化视觉传输场景：

#### 2.1.1 定位标记 (Finder Patterns)

`
[14×14大标记]          [中心十字]        [7×7小标记]
   (0,0)               (72,72)            (137,137)
    ●●●                   |                 ●●
   ●   ●                ─┼─                ● ●
   ● ● ●                  |                 ●●
`

- **3个大型定位标记** (14×14): 左上、右上、左下角
- **1个小型定位标记** (7×7): 右下角
- **中心对齐十字** (5×5): 辅助透视校正

**设计理由**:
- 不对称布局可识别旋转方向（避免180°混淆）
- 大标记在远距离或模糊条件下仍可识别
- 中心十字提供第5个控制点用于镜头畸变校正

#### 2.1.2 信息头 (Header Region)

**位置**: (2,10) ~ (18,26) 矩形区域  
**容量**: 64 bits  
**布局**:

| 字段        | 位数 | 说明                                |
|-------------|------|-------------------------------------|
| CRC32       | 32   | 当前帧数据的循环冗余校验码          |
| data_len    | 16   | 数据有效字节数 (0-2183)             |
| frame_idx   | 16   | 帧序号 (0-65535)                    |

**读取顺序**: 从左到右、从上到下的光栅扫描

#### 2.1.3 数据区域 (Data Region)

**位置**: (2,106) ~ (27,98) 矩形区域  
**容量**: 18,748 bits (2,343 bytes 物理容量)  
**净容量**: 17,464 bits (2,183 bytes 数据 + 160 bytes ECC)

**蛇形扫描布局** (Snake Scan):
`
行0: 0→1→2→3→...→105  (从左到右)
行1: 105←104←...←1←0  (从右到左)
行2: 0→1→2→3→...→105  (从左到右)
...
`

**设计理由**: 
- 蛇形扫描保证相邻数据位在空间上也相邻
- 有利于局部错误的空间聚集，提升RS纠错效率
- 减少行间跳变时的同步错误

### 2.2 Reed-Solomon 纠错编码

#### 2.2.1 编码参数

`python
rs = reedsolo.RSCodec(nsym=16, nsize=255)
`

- **nsym**: 16字节纠错符号，可纠正 ≤8 字节错误
- **nsize**: GF(2^8)域大小，最大块尺寸255字节
- **最大数据块**: 255 - 16 = 239字节

#### 2.2.2 分块编码策略

对于2,183字节的数据：
`
Block 0: 239 bytes data → 255 bytes (239 + 16 ECC)
Block 1: 239 bytes data → 255 bytes (239 + 16 ECC)
...
Block 8: 239 bytes data → 255 bytes (239 + 16 ECC)
Block 9: 32 bytes data  → 48 bytes (32 + 16 ECC)

Total: 2,183 data bytes + 160 ECC bytes = 2,343 bytes
`

**容量计算公式** (二分搜索实现):
`python
def calc_ecc_size(data_bytes):
    return len(rs.encode(b'\x00' * data_bytes))

# 二分查找: 找到最大 data_bytes 使得
# calc_ecc_size(data_bytes) ≤ PHYSICAL_CAPACITY (2,343)
DATA_SIZE_LIMIT = 2,183 bytes
`

## 3. 核心技术实现

### 3.1 编码流程 (src/_2Dcode.py)

#### 3.1.1 核心函数: ncode_bin(data: bytes) -> np.ndarray

`python
def encode_bin(data: bytes) -> np.ndarray:
    """
    将二进制数据编码为二维码矩阵
    
    Args:
        data: 原始字节数据 (≤2,183 bytes)
    
    Returns:
        144×144 numpy数组，元素值0或255
    """
    # 1. 计算CRC32校验
    crc32_val = zlib.crc32(data) & 0xFFFFFFFF
    
    # 2. RS编码
    encoded_data = rs.encode(data)
    
    # 3. 构建信息头
    header = pack_bits([
        crc32_val,      # 32 bits
        len(data),      # 16 bits
        frame_index     # 16 bits
    ])
    
    # 4. 填充到BASE_GRID
    grid = BASE_GRID.copy()
    fill_region(grid, INFO_ITER, header)
    fill_region(grid, DATA_ITER, encoded_data)
    
    return grid
`

#### 3.1.2 容量计算优化 (Lines 190-219)

**问题**: 早期版本直接使用 DATA_SIZE_LIMIT = 物理容量 - 16，未考虑RS分块  
**影响**: 大文件编码时数据溢出，导致截断或崩溃

**修复方案**:
`python
def find_max_data_size(physical_capacity):
    """二分查找最大可编码数据量"""
    left, right = 0, physical_capacity
    
    while left < right:
        mid = (left + right + 1) // 2
        test_size = rs.encode(b'\x00' * mid)
        
        if len(test_size) <= physical_capacity:
            left = mid
        else:
            right = mid - 1
    
    return left

DATA_SIZE_LIMIT = find_max_data_size(2343)  # 结果: 2,183
`

### 3.2 解码流程 (src/_2Dcode.py)

#### 3.2.1 核心函数: decode_image(img: np.ndarray) -> bytes

`python
def decode_image(img: np.ndarray) -> bytes:
    """
    从二维码图像恢复数据
    
    Args:
        img: 144×144灰度图像 (或更大，会自动缩放)
    
    Returns:
        解码后的原始字节数据
    """
    # 1. 图像预处理
    if img.shape != (144, 144):
        img = cv2.resize(img, (144, 144), interpolation=cv2.INTER_NEAREST)
    
    # 2. 二值化 (自适应阈值)
    binary = (img > 128).astype(np.uint8) * 255
    
    # 3. 提取信息头
    header_bits = extract_region(binary, INFO_ITER)
    crc32_val = bits_to_int(header_bits[0:32])
    data_len = bits_to_int(header_bits[32:48])
    frame_idx = bits_to_int(header_bits[48:64])
    
    # 4. 提取数据区
    data_bits = extract_region(binary, DATA_ITER)
    
    # 5. 计算RS编码总长度 (修复后的算法)
    test_encoded = rs.encode(b'\x00' * data_len)
    total_byte_len = len(test_encoded)  # 精确长度
    
    # 6. 转换为字节
    encoded_bytes = bits_to_bytes(data_bits[:total_byte_len * 8])
    
    # 7. RS解码
    decoded_data = rs.decode(encoded_bytes)[0]
    
    # 8. CRC校验
    if zlib.crc32(decoded_data) != crc32_val:
        raise ValueError("CRC mismatch")
    
    return decoded_data
`

#### 3.2.2 ECC长度计算修复 (Lines 402-409)

**问题**: 原始代码使用 	otal_len = data_len + 16  
**影响**: 多块数据时总长度错误，RS解码收到不完整数据  
**错误示例**:
`python
# 错误: data_len=2183, 计算为2183+16=2199
# 实际: 需要2343字节 (10个块，每块16 ECC)
`

**修复**:
`python

`

### 3.3 视频生成 (src/utils/bin2video.py)

#### 3.3.1 上采样策略

`python
def upsample_qr(qr_matrix):
    """10倍上采样: 144×144 → 1440×1440"""
    return cv2.resize(
        qr_matrix,
        (1440, 1440),
        interpolation=cv2.INTER_NEAREST  # 最近邻，保持锐利边缘
    )
`

**为什么10倍?**
- 确保单个模块至少10×10像素
- 相机分辨率1080p时，每个模块约7.5像素（安全余量）
- MP4压缩后边缘仍清晰可辨

#### 3.3.2 视频编码参数

`python
fourcc = cv2.VideoWriter_fourcc(*'mp4v')  # MPEG-4编码
fps = 15  # 每秒15帧 (相机易于捕获的速度)
resolution = (1440, 1440)
`

**帧率选择依据**:
- 手机相机通常30fps，15fps给予2倍时间余量
- 更低帧率(如5fps)可能导致屏幕刷新与相机不同步
- 更高帧率(如30fps)增加丢帧风险

## 8. 问题修复记录

### 8.1 Bug #1: 容量计算错误

**发现时间**: 2026-03-18  
**严重性**: 🔴 Critical

**问题描述**:
编码200KB文件时程序崩溃，错误信息为 IndexError: list index out of range

**根本原因**:
`python
# 错误代码 (旧版 _2Dcode.py:195)
DATA_SIZE_LIMIT = len(DATA_ITER) // 8 - ECC_BYTES
# 计算为: 18748 // 8 - 16 = 2327 bytes

# 实际情况:
rs.encode(b'\x00' * 2327) → 长度2503字节 > 物理容量2343
`

RS编码器使用分块策略，每239字节数据添加16字节ECC，总开销远大于16字节。

**修复方案**:
实现二分搜索算法，测试实际编码长度:

`python
def find_max_data_size(physical_capacity):
    left, right = 0, physical_capacity
    while left < right:
        mid = (left + right + 1) // 2
        if len(rs.encode(b'\x00' * mid)) <= physical_capacity:
            left = mid
        else:
            right = mid - 1
    return left

DATA_SIZE_LIMIT = find_max_data_size(2343)  # 结果: 2183
`

**Git提交**: 5fd01d5 - Fixed capacity calculation bug

### 10.2 鲁棒性增强

#### 10.2.1 透视校正
**问题**: 斜角拍摄导致QR码变形  
**方案**: 利用4个定位标记进行透视变换

`python
def perspective_correct(img, corners):
    src_pts = np.array(corners, dtype=np.float32)
    dst_pts = np.array([
        [0, 0], [144, 0], 
        [144, 144], [0, 144]
    ], dtype=np.float32)
    
    M = cv2.getPerspectiveTransform(src_pts, dst_pts)
    return cv2.warpPerspective(img, M, (144, 144))
`
